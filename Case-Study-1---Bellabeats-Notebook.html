Summary

About The Company

Bellabeat, a high-tech company founded by Urška Sršen and Sando Mur that manufactures beautifully designed health-focused smart products.
The particularly unique aspect of this smart-technology is the fact that it informs and empowers women around the world by collecting consumer data on activity, sleep, stress, and reproductive health. This allows Bellabeat to empower women with knowledge regarding their own health and habits.

Sršen, co-founder and Chief Creative Officer of Bellabeat, believes that an analysis of smart device fitness data will help gain insight on how consumers are using their smart devices, which could reveal more opportunities for growth in their marketing strategy.

Business Task

Analyze smart device fitness data to identify potential opportunities for growth and provide high-level content recommendations for improvement on the Bellabeat marketing strategy based on the analysis.

Data Source

Smart Device Fitness Data

For this study, I will use the FitBit Fitness Tracker Data conducted by Kaggle user Arashnic.

This dataset is particularly interesting to me as a Healthcare Analyst because the creator’s inspiration was “Human temporal routine behavioral analysis and pattern recognition”, which is a topic I am highly fascinated by.

Dataset Content

The FitBit Fitness Tracker data set is generated by thirty eligible Fitbit users who consented to participating in a survey between March 12, 2016 to May 12, 2016 via Amazon Mechanical Turk. Participants included the submission of personal tracker data, including minute-level output for physical activity, heart rate, and sleep monitoring.

The original data set creator mentions that “Variation between output represents use of different types of Fitbit trackers and individual tracking behaviors / preferences.“

Questions for Analysis

Process

Data Transformation, and Manipulation

After running these code chunks to summarize data, I noticed that the data will require cleaning:

  • The Date and Time are combined in one column which can be a problem during our analysis later on if we don’t separate it.
  • To better capture the changes over time, I’ll convert all the hourly time values to minutes for a clearer comparison.
  • I’ve noticed that the variable names for dates are inconsistent across some of the data frames. I’ll need to rename them and ensure the date strings are properly formatted for consistency.

daily_activity:

  • needs to be lowercase.
  • activitydate will be renamed to “date” for consistency.
  • select all values except calories, which belongs in the calories data frame.
activity <- rename_with(daily_activity, tolower) %>%
  rename(date=activitydate) %>% 
  mutate(date = as.Date(date, format="%m/%d/%Y"))

daily_intensities:

  • needs to be lowercase
  • activityday needs to be renamed to “date” for consistency
  • date needs to be in date format
  • for analysis later on, I will also convert the minutes variables into hours
intensities <- rename_with(daily_intensities, tolower) %>%   
  rename(date=activityday) %>%   
  mutate(date = as.Date(date, format="%m/%d/%Y"))

daily_steps:

  • needs to be lowercase
  • format activityday to date type
  • rename activityday to “date” for consistency
  • rename stepstotal to “totalsteps” for consistency with activity df
steps <- rename_with(daily_steps, tolower) %>%
  rename(date=activityday) %>% 
  rename(totalsteps=steptotal) %>% 
  mutate(date = as.Date(date, format="%m/%d/%Y"))

daily_calories:

  • needs to be lowercase
  • activityday needs to be renamed to “date” for consistency * date needs to be in date format
calories <- rename_with(daily_calories, tolower) %>% 
  rename(date=activityday) %>% 
  mutate(date = as.Date(date, format="%m/%d/%Y"))

sleep_day:

  • needs to be lowercase
  • sleepday needs to be renamed to “date” for consistency
  • date needs to be in date format
sleep <- rename_with(sleep_day, tolower) %>% 
  separate(sleepday, into=c("date", "time"), sep=" ") %>% 
  mutate(date = as.Date(date, format="%m/%d/%Y"))

weight_log_info:

  • needs to be lowercase
  • date needs to be separated to “date” and “time”
  • date needs to be in date format * time needs to be in time format
weightlog <- rename_with(weight_log_info, tolower) %>% 
  separate(date, into=c("date", "time"), sep=" ") %>% 
  mutate(date = as.Date(date, format="%m/%d/%Y"))

heart_rate:

  • needs to be lowercase
  • time needs to be separated to “date” and “time”
  • date needs to be in date format
  • time needs to be in time format
heartrate <- rename_with(heart_rate, tolower) %>% 
  mutate(datetime=mdy_hms(time))

Q: What percentage of the participants use each of the categories?

Game Plan:

To easily view and compare, we will create a new data frame named “participants”.

Participants df will include: The names of each category tracked with the FitBit, number of participants out of 33 that used each category, and that number in a percentage.

  • Create a new vector of all the categories the FitBit can track and name it “category”.

  • Create another vector with the unique number of participants we named earlier.

  • Create another column named “participant_percentage” – There are 33 unique participants in total. To obtain the percentage, we need to divide the number_participants in each category, by the total number of participants (33), and then multiply by 100.

  • Create a new data frame with all the vectors/columns we just created.

category <- c("activity", "calories", "intensities", "sleep", "steps", "weightlog", "heartrate")

number_participants <- c(activity_participants, calories_participants, intensities_participants, sleep_participants, steps_participants, weightlog_participants, heartrate_participants)

participants <- data.frame(category, number_participants, participant_percentage)

Q: What is the relationship between steps taken in day and calories?

Hypothesis: The relationship between steps taken in a day (totalsteps) and calories (calories) should be a positive correlation. More steps taken in a day should result in a higher amount of calories burned.

ggplot(data=activity) +
  geom_point(mapping=aes(x=totalsteps, y=calories)) +
  geom_smooth(mapping=aes(x=totalsteps, y=calories)) +
  labs(title="Total Active Minutes vs. Calories", caption="This plot demonstrates the relationship between total steps taken and calories burned.") +
  xlab("Total Steps Daily") +
  ylab("Calories Burned Daily") +
  theme(plot.title=element_text(size=20))

Summary of Analysis:

The plot ‘Total Active Minutes vs Calories’ plot above clearly demonstrates a Positive Linear Correlation between steps taken in a day and calories burned.

This suggests that our hypothesis is correct - The more steps we take in a day, the more calories we burn.

Insights for Bellabeat

  • As an enhanced feature, Bellabeat could introduce an in-app goal-setter.

  • Goal-setting activity could take the form of a challenge, encouraging users to meet daily step goals to maximize calorie burn.

  • Bellabeat app could include a dedicated page with recommendations for calculating the amount of time users should spend being active to achieve their calorie-burning goals.
